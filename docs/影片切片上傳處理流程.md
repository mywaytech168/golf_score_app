### 2.4 並列工作隊列

**技術選擇**：
- 推薦：**Hangfire**（.NET 內置任務隊列）或 **HostedService**
- 替代：RabbitMQ、Redis Queue

**實現邏輯**：
```csharp
public class VideoProcessingService : BackgroundService
{
    private readonly int _workerCount = 3;  // 並列度
    private readonly ILogger<VideoProcessingService> _logger;
    private readonly IVideoProcessingRepository _repo;

    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        var tasks = new List<Task>();
        
        // 啟動 N 個 Worker
        for (int i = 0; i < _workerCount; i++)
        {
            tasks.Add(ProcessQueueWorker(stoppingToken));
        }
        
        await Task.WhenAll(tasks);
    }

    private async Task ProcessQueueWorker(CancellationToken stoppingToken)
    {
        while (!stoppingToken.IsCancellationRequested)
        {
            // 從隊列取出一個待處理的切片
            var queueItem = await _repo.GetNextQueuedSliceAsync();
            if (queueItem == null)
            {
                await Task.Delay(1000, stoppingToken);  // 沒有任務時等待 1 秒
                continue;
            }

            try
            {
                // 標記為處理中
                await _repo.UpdateSliceStatusAsync(queueItem.SliceId, "processing");

                // 呼叫 Python 處理
                var result = await ProcessVideoWithPythonAsync(queueItem);

                if (result.Success)
                {
                    // 標記為已完成
                    await _repo.UpdateSliceStatusAsync(queueItem.SliceId, "completed");
                    await _repo.SaveOutputFilesAsync(queueItem.SliceId, result.OutputFiles);
                }
                else
                {
                    // 標記為失敗
                    await _repo.UpdateSliceStatusAsync(queueItem.SliceId, "failed");
                    await _repo.SaveErrorLogAsync(queueItem.SliceId, result.Error);
                }
            }
            catch (Exception ex)
            {
                _logger.LogError(ex, "Error processing slice {SliceId}", queueItem.SliceId);
                await _repo.UpdateSliceStatusAsync(queueItem.SliceId, "debug");
            }
        }
    }
}
```

**狀態轉移圖**：
```
pending
   ↓
[隊列等待]
   ↓
processing (Worker 執行)
   ↓
   ├─→ completed ✅
   ├─→ failed ❌ → (可重試)
   └─→ debug 🔧 (手動處理)
```

### 2.6 錯誤處理與重試

**重試策略**：
```csharp
public class RetryPolicy
{
    public static async Task<T> ExecuteWithRetryAsync<T>(
        Func<Task<T>> operation,
        int maxRetries = 3,
        int delayMilliseconds = 5000)
    {
        for (int attempt = 1; attempt <= maxRetries; attempt++)
        {
            try
            {
                return await operation();
            }
            catch (Exception ex) when (attempt < maxRetries)
            {
                _logger.LogWarning(ex, $"Attempt {attempt} failed, retrying in {delayMilliseconds}ms...");
                await Task.Delay(delayMilliseconds);
            }
        }
        throw new InvalidOperationException($"Operation failed after {maxRetries} attempts");
    }
}
```

**錯誤標記**：
- ✅ 一般錯誤 → 標記 `failed`，可自動重試
- ✅ 檔案缺失 → 標記 `debug`，需手動處理
- ✅ Python 異常 → 標記 `debug`，記錄詳細錯誤信息

## 📝 備註與常見問題

### Q1：本地切片後，如何同步到伺服器？
**A**：前端維護一個 `sync_status` 欄位：
- `local_only`：僅本地存在
- `synced`：已上傳並確認伺服器接收
- `out_of_sync`：伺服器版本更新，本地未同步

### Q2：如何支持斷點續傳？
**A**：使用分塊上傳：
- 將大檔案分成 5MB 的塊
- 上傳前檢查伺服器是否已有部分塊
- 只上傳缺失的塊

### Q3：Python 處理失敗怎麼辦？
**A**：
1. 自動重試 3 次（延遲遞增）
2. 記錄詳細錯誤信息
3. 標記為 `debug` 狀態
4. 提供手動重試按鈕

### Q4：如何控制並列度？
**A**：在 C# Server 配置：
```csharp
private readonly int _workerCount = Environment.GetEnvironmentVariable("WORKER_COUNT") 
    ?? "3";  // 預設 3 個並列 Worker
```
