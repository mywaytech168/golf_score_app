# 錄製對象分流藍圖：自己 vs 其他人

> 日期：2026-06-15
> 範圍：錄影模式（`record_screen.dart`）+ SHOT 模式（`shot_record_screen.dart`）
> 狀態：規劃中（未實作）

---

## 1. 核心判斷

「自己 vs 其他人」改變的是**互動／觸發模型**與**取景輔助**，**不是分析管線**。

骨架推論、擊球判定（V1 弧底 / V4 錨點）、雙手判斷、自動切片、P-System、音訊評分等
全部與「拍誰」無關，**一律不動**。subject 只影響三件事：

1. **觸發模型** — 誰決定開錄／停錄的時機
2. **取景輔助** — 預設鏡頭、開錄前倒數、提示文案
3. **節奏控制** — SHOT 模式是否自動進入下一桿

### 現況對照

| 現有模式 | 互動模型 | 隱含對象 | 痛點 |
|---|---|---|---|
| 錄影模式 | 手動 tap 開始/停止 | 別人幫錄 / 自己按了再就位 | **自己一個人**：tap 後要跑到定位（前段廢片）、結束跑回按停 |
| SHOT 模式 | 站姿自動偵測 → 自動開錄 → 自動下一桿 | 自己一個人練習 | **幫別人錄**：卡在站姿偵測才開錄；朋友握桿/角度易偵測不準；操作者想自己抓時機 |

---

## 2. 行為矩陣（已確認設計）

| 面向 | 自己 (self) | 其他人 (other) |
|---|---|---|
| **錄影模式 — 開始** | tap → 倒數 N 秒（可設 0/3/5/10）就位 → 開錄 | tap → 立即開錄（現狀） |
| **錄影模式 — 停止** | 手動（維持現狀；自動停止列 backlog） | 手動（現狀） |
| **錄影模式 — 預設鏡頭** | 後鏡頭(腳架)，可切前鏡頭自我取景 | 後鏡頭 |
| **SHOT 模式 — 觸發** | 站姿自動偵測（現狀，免手） | **操作者按「開始」立即進入錄製，跳過站姿等待** |
| **SHOT 模式 — 自動下一桿** | 開（連續自練，現狀） | **關**（操作者控節奏，打完回 idle 等下一次按） |
| 擊球/切片/光暈/錨點/雙手 | 不變 | 不變 |
| 提示音/震動不入軌策略 | 不變 | 不變 |

> 決策依據（2026-06-15 與使用者確認）：
> - SHOT 其他人 = **操作者手動開始**（最貼近實際幫拍情境），非保留站姿自動偵測
> - 範圍 = **完整藍圖照做**，兩模式都改

## 3. 實作落點（沿用既有 Prefs / Settings 模式，低風險）

### Phase 0 — 資料與管線

- 新 `enum RecordSubject { selfUser, other }`（放 `lib/recording/recording_config.dart` 或新 `record_subject.dart`）
- `SwingDetectPrefs` 加：
  - `getRecordSubject() / setRecordSubject()`（key `record_subject`，預設 `selfUser`，記住上次選擇）
  - `getSelfCountdownSec() / setSelfCountdownSec()`（key `self_countdown_sec`，預設 5，clamp 0~10）
- `RecordScreen` / `ShotRecordScreen` 建構子加 `RecordSubject subject`，由 selection screen 帶入
- 兩頁 `initState`：`_isFront` 初值依 subject（self 可保留前鏡頭選項；預設仍後鏡頭）

### Phase 1 — Selector（入口）

- `RecordingSelectionScreen` 頂部（header 下方）加 **SegmentedButton**「要錄誰？ 自己 / 其他人」
- 選擇寫入 `SwingDetectPrefs.setRecordSubject`，並傳入 `_startShotMode()` / `_startRecording()`
- 兩錄製頁 settings sheet 內各加一列快速切換（與其他開關同風格）

### Phase 2 — 錄影模式 self 倒數

- `record_screen.dart` `_startRecording()`：
  - `subject == self && countdown > 0` 時，先顯示全螢幕大倒數 overlay（沿用 SHOT 的 `_promptChip` 風格大字）
  - 倒數結束才呼叫 `_camera.startRecording`（prewarm session 不受影響，倒數在 startRecording **之前**）
  - 倒數期間可取消（返回 / 點螢幕）
- settings 加「就位倒數秒數」滑桿（0/3/5/10，0 = 立即）

### Phase 3 — SHOT 模式 other 手動路徑

- `shot_record_screen.dart`：
  - `subject == other` 時 `_startShot()` 後**不進 addressing 等站姿**，直接走 `_beginRecording()` 路徑
    - 注意 `_beginRecording` 內 `_addressConfirmed` 相關旗標與 `_handleImpact` 的 `!_addressConfirmed` 早退守門 → other 模式需把 `_addressConfirmed` 視為 true（手動開始即視同確認）
    - 站姿提示音 `playAddressConfirmedAndWait()` 在 other 模式可省略（無站姿確認語意）
  - idle overlay：other 模式「準備」鈕文案改「開始」，按下立即錄製
  - `_finishShotRecording()` 結尾：`subject == other` 時**不**啟動 `_startAutoNextCountdown()`，直接停在 result，等操作者按「下一桿」
  - recording overlay 維持手動「停止」鈕（現狀已有）

### Phase 4 — 收尾

- i18n 四語系（zh_TW / zh / zh_CN / en）：
  - `recSubjectTitle`（要錄誰？）、`recSubjectSelf`（自己）、`recSubjectOther`（其他人）
  - `recSelfCountdown`（就位倒數）、倒數 overlay 文案、SHOT other 「開始」鈕文案
- `flutter analyze 0`、`flutter test` 全綠
- 與 `docs/分析系統升級藍圖_2026-06-14.md` 對齊歸檔，更新 status.md / changelog.md

## 4. 風險與注意

- **prewarm 不可被破壞**：倒數只在 `startRecording` 前等待，不重做 `prepareForRecording`，避免覆蓋 native preparedRecPath（既有錄製流程鐵律）。
- **SHOT other 的偵測器校準**：原校準在 addressing 期完成。other 模式跳過 addressing → 需確認 `LiveSwingDetector` 在 `_beginRecording` 後仍有足夠校準窗（前 ~3 秒），或在開始錄製後保留校準期。**這是 Phase 3 最需實機驗證的點。**
- **`_addressConfirmed` 守門**：`_handleImpact` 用它擋掉站姿確認前的誤觸；other 手動開始需正確設為已確認，否則第一桿不被記錄。
- **鏡頭預設**：self 仍預設後鏡頭（腳架自拍主流），前鏡頭僅為「站近取景」選項，不強制。
- **跨平台分享**：`recordedPlatform` / 前鏡頭 flip 既有邏輯不受 subject 影響（subject 不入庫，純錄製期行為）。

---

## 5. 不做（明確排除）

- 不動任何分析演算法 / 切片 / P-System / 上傳路徑
- 不為 subject 在 `RecordingHistoryEntry` 增欄位（純錄製期 UX，不需持久化到切片）
- 錄影模式「自動停止」（偵測 finish 後 X 秒收尾）列 backlog，非本批；自己一個人要全自動就用 SHOT 模式
- iOS 原生無需改動（純 Dart 層 UX 分流）
