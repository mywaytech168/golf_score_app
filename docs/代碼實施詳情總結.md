#### 📍 1. LocalSliceRepository (`lib/services/local_slice_repository.dart`)

**用途**：管理本地 SQLite 資料庫，存儲切片元資料和狀態

**核心功能**：

```dart
// 初始化資料庫
Future<Database> _initializeDatabase()

// 插入紀錄
Future<void> insertRecording(...)
Future<void> insertSlice(...)
Future<void> insertSlicesBatch(List<Map> slices)

// 更新狀態
Future<void> updateSliceStatus(String sliceId, String status)
Future<void> markSliceAsUploaded(String sliceId, int serverId)
Future<void> syncSliceStatus(String sliceId, String serverStatus)

// 查詢方法
Future<List<Map>> getPendingSlices(String recordingId)
Future<List<Map>> getSlicesByRecording(String recordingId)
Future<Map> getRecordingStats(String recordingId)

// 管理方法
Future<void> deleteRecording(String recordingId)
Future<void> clearDatabase()
```

**資料庫模式**：

```sql
-- 本地錄影紀錄表
CREATE TABLE local_recordings (
  id TEXT PRIMARY KEY,
  name TEXT,
  created_at DATETIME,
  total_slices INTEGER,
  sync_status TEXT  -- 'local_only', 'synced', 'out_of_sync'
);

-- 本地切片紀錄表
CREATE TABLE local_slices (
  id TEXT PRIMARY KEY,
  recording_id TEXT,
  slice_index INTEGER,
  video_file_path TEXT,
  trajectory_csv_path TEXT,
  status TEXT,  -- 'pending', 'uploaded', 'processing', 'completed', 'failed'
  upload_time DATETIME,
  server_id TEXT,  -- 對應伺服器的 slice_id
  sync_timestamp DATETIME
);
```

## 🔌 前後端連接點

### 前端調用流程

```
用戶點擊「上傳」
    ↓
LocalSliceManagementPage._uploadSelectedSlices()
    ↓
_UploadProgressDialog.startUpload()
    ↓
for each slice:
  - VideoServerClient.uploadSlice()
    ↓
  - HTTP POST /api/slices/upload
    ↓
  - 接收 server_id
    ↓
  - LocalSliceRepository.markSliceAsUploaded()
    ↓
  - 更新本地 SQLite
    ↓
  - 更新 UI 進度
```

### 伺服器端處理流程

```
HTTP POST /api/slices/upload
    ↓
VideoController.UploadSlice()
    ↓
驗證請求
    ↓
VideoUploadService.UploadSliceAsync()
    ↓
1. 建立目錄結構
2. 儲存影片檔
3. 儲存 CSV 檔
4. 建立 Slice 紀錄
    ↓
返回 200 + SliceId
    ↓
[等待] BackgroundService 檢測新切片
    ↓
更新狀態: pending → processing
    ↓
呼叫 Python 處理
```
