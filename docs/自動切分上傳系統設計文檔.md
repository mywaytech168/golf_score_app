# 自動切分與上傳系統設計文檔

## 概述

本系統實現了**自動視頻切分完成後立即上傳**的完整工作流程，包括元數據上傳、視頻文件上傳和 CSV 數據上傳的協調管理。

## 核心架構

### 1. 上傳隊列系統（SwingClipUploadManager）

**文件**: `lib/services/swing_clip_upload_manager.dart`

#### 核心概念

```
SwingClipResult (切分結果)
        ↓
UploadQueueItem (隊列項)
        ↓
[Queue] → [Processing] → [Upload]
  ├─ Metadata
  ├─ Video File
  └─ CSV Data
```

#### UploadQueueItem 結構

```dart
class UploadQueueItem {
  final String id;              // 唯一 ID
  final String recordingId;     // 原始錄影 ID
  final String videoPath;       // 切片視頻路徑
  final String? csvPath;        // 切片 CSV 路徑
  final Map metadata;           // 切片元數據
  UploadStatus uploadStatus;    // 上傳狀態
  double uploadProgress;        // 上傳進度 (0.0-1.0)
  String? uploadError;          // 失敗原因
  int? clipId;                  // 伺服器返回的 ID
  int retryCount;               // 重試次數
}
```

#### 主要方法

| 方法 | 功能 |
|------|------|
| `addClipToQueue()` | 新增單個切片到隊列 |
| `addClipsToQueue()` | 批量新增切片 |
| `startProcessing()` | 開始上傳隊列 |
| `pauseProcessing()` | 暫停上傳 |
| `resumeProcessing()` | 繼續上傳 |
| `cancelAll()` | 取消所有 |
| `getStatistics()` | 取得統計信息 |

### 2. 自動切分與上傳服務（AutoSplitAndUploadService）

**文件**: `lib/services/auto_split_and_upload_service.dart`

#### 完整工作流

```
Input: 原始視頻 + IMU CSV
  ↓
1️⃣  SwingSplitService.split()
     └─ 檢測擊棒峰值
     └─ 切割視頻片段
     └─ 提取 CSV 子集
  ↓
2️⃣  添加到上傳隊列
     └─ 為每個切片建立 UploadQueueItem
     └─ 包裝元數據
  ↓
3️⃣  啟動自動上傳
     └─ SwingClipUploadManager.startProcessing()
     └─ 逐個上傳項目
  ↓
Output: 上傳完成的切片清單
```

#### 方法簽名

```dart
Future<int> executeAutoSplitAndUpload({
  required String videoPath,       // 原始視頻
  required String imuCsvPath,      // IMU 數據
  required String recordingId,     // 錄影 ID
  double windowBeforeSec = 3.0,    // 前置時間窗口
  double windowAfterSec = 1.0,     // 後置時間窗口
  double threshG = 20.0,           // 加速度閾值
  bool autoStartUpload = true,     // 自動上傳
})
```

### 3. 後端上傳端點（SwingClipController）

**文件**: `server/Controllers/SwingClipController.cs`

#### 三步上傳流程

```
Step 1: 上傳元數據
  POST /api/clips/metadata
  Body: { "tag", "hitSecond", "peakValue", ... }
  Response: { "clipId": 123, "uploadUrl": "..." }
  
Step 2: 上傳視頻文件（分塊）
  POST /api/clips/{clipId}/upload
  Body: multipart/form-data (視頻分塊)
  Headers: X-Chunk-Index, X-Total-Chunks, Content-Range
  Response: { "isComplete": true/false, "progress": 0.8 }
  
Step 3: 上傳 CSV 數據
  POST /api/clips/{clipId}/csv
  Body: multipart/form-data (CSV 文件)
  Response: { "savedPath": "..." }
```

#### 端點清單

| 方法 | 端點 | 功能 |
|------|------|------|
| POST | `/api/clips/metadata` | 建立切片元數據 |
| POST | `/api/clips/{id}/upload` | 上傳視頻文件 |
| POST | `/api/clips/{id}/csv` | 上傳 CSV 數據 |
| POST | `/api/clips/{id}/complete` | 標記上傳完成 |
| GET | `/api/clips/{id}` | 查詢切片詳情 |
| GET | `/api/clips/recording/{rid}` | 查詢錄影的切片 |
| DELETE | `/api/clips/{id}` | 刪除切片 |
| POST | `/api/clips/delete-batch` | 批量刪除 |
| GET | `/api/clips/stats` | 統計信息 |

## 實施流程

### 場景 1：錄製完成後自動切分上傳

```dart
// 在錄製完成後調用
final service = AutoSplitAndUploadService(
  uploadManager: uploadManager,
  onStatusChanged: (status) {
    setState(() => _status = status);
  },
  onSplitProgress: (progress) {
    setState(() => _splitProgress = progress);
  },
  onUploadProgress: (progress) {
    setState(() => _uploadProgress = progress);
  },
);

final clipCount = await service.executeAutoSplitAndUpload(
  videoPath: '/path/to/video.mp4',
  imuCsvPath: '/path/to/imu.csv',
  recordingId: recordingHistoryEntry.filePath,
  autoStartUpload: true,
);

print('成功上傳 $clipCount 個切片');
```

### 場景 2：手動管理上傳隊列

```dart
// 新增單個切片
uploadManager.addClipToQueue(
  recordingId: 'rec_123',
  videoPath: '/path/to/clip_1.mp4',
  csvPath: '/path/to/clip_1.csv',
  metadata: {
    'tag': 'swing_1',
    'hitSecond': 5.5,
    'goodShot': true,
  },
);

// 開始上傳
await uploadManager.startProcessing();

// 暫停
uploadManager.pauseProcessing();

// 繼續
await uploadManager.resumeProcessing();

// 取得統計
final stats = uploadManager.getStatistics();
print('待上傳: ${stats['local']}, 已上傳: ${stats['uploaded']}');
```

## 上傳狀態轉移

```
┌─────────────────────────────────────────────┐
│                                             │
│         [Local] ─────初始化─────┐           │
│            │                     ↓           │
│            │                 [Uploading]     │
│            │                  ↙  ↖           │
│            │              ✓ /      \ ✗       │
│            │             /          \       │
│            ↓            ↓            ↓      │
│         [Uploaded]                [Failed]  │
│            ✅              重試 ↓    ↑       │
│                              [Uploading]    │
│                                             │
└─────────────────────────────────────────────┘
```

### 狀態說明

| 狀態 | 含義 | 可進行操作 |
|------|------|----------|
| `local` | 待上傳 | 上傳、刪除 |
| `uploading` | 上傳中 | 暫停、取消 |
| `uploaded` | 已完成 | 查看、刪除 |
| `failed` | 上傳失敗 | 重試、刪除、查看錯誤 |

## 關鍵特性

### 1. 分塊上傳
```
大文件 (500MB) → [分塊] (每塊 50MB) → [逐塊上傳]
                          ↓
                  支持斷點續傳 & 並行上傳
```

### 2. 優先級隊列
```
元數據優先 → 視頻文件 → CSV 數據
   (立即)    (5MB/塊)   (最後)
```

### 3. 自動重試
```
失敗 → [等待 5 秒] → 重試
               ↑
          最多 3 次
```

### 4. 進度追蹤
```
元數據:   0%-20%
視頻:    20%-80%
CSV:     80%-100%
```

## 效能考慮

```
UploadQueueItem: ~1KB
100 個切片隊列: ~100KB
+視頻文件流: ~5-10MB (分塊上傳時)
```

### 網絡優化
```
分塊大小: 5MB (平衡速度與穩定性)
重試延遲: 5 秒
超時限制: 30 秒
```

### 併發控制
```
同時上傳項目: 1 個 (保證順序)
分塊並發: 3 個 (加快上傳)
```

## 常見問題

**Q: 上傳中斷後能否繼續？**
A: 可以。隊列持久化到本地 JSON，重啟應用時自動恢復未完成的上傳。

**Q: 大文件上傳需要多長時間？**
A: 100MB 視頻在 5Mbps 網絡下約需 160 秒。系統支持並行上傳加快速度。

**Q: 上傳失敗後怎麼辦？**
A: 系統自動重試最多 3 次。失敗項目的 `uploadError` 欄位記錄原因，用戶可查看和手動重試。

**Q: 能否暫停上傳省流量？**
A: 可以。調用 `pauseProcessing()` 暫停，待網絡恢復後調用 `resumeProcessing()` 繼續。
