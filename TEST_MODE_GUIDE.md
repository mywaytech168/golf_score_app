# 录影测试模式使用指南

## 📋 功能概述

录影页面现已支持**两种模式**：

### 1. 🎥 即时录制模式（Real-time Recording）
- **默认模式**：使用设备摄像头进行实时高尔夫挥杆录制
- **操作**：点击底部录制按钮开始/结束录制
- **输出**：录制视频、姿态数据CSV、音频文件

### 2. 🧪 测试模式（Test Mode）
- **新增功能**：从已导入的视频中选择一个作为"测试录制"
- **用途**：用于演示、测试和无需实时录制的快速验证
- **操作**：切换到测试模式 → 选择视频 → 完成

---

## 🚀 使用方法

### 启用测试模式

1. **打开录影页面**
   - 在应用中导航到录影/录制屏幕

2. **切换到测试模式**
   - 在右上角找到 `🎥 即时` 按钮
   - 点击切换到 `🧪 测试` 模式
   - 按钮会变为**橙色**，表示已激活测试模式

### 选择测试影片

1. **点击"选择影片"按钮**
   - 从已导入的视频列表中选择
   - 显示视频名称、时长和Round编号

2. **确认选择**
   - 视频信息会显示在屏幕中心
   - 支持"取消"返回重新选择

### 完成测试录制

1. **点击"完成"按钮**
   - 系统会自动复制选定的视频
   - 生成对应的CSV和音频文件
   - 返回到视频处理流程（与实时录制相同）

## ⚙️ 技术实现细节

### 测试模式的核心逻辑 (`_completeTestMode`)

```dart
1. 创建测试工作目录：`golf_recordings/test_{timestamp}/`
2. 复制源视频文件到测试目录
3. 生成标准CSV文件（仅包含头部）
4. 创建占位符音频文件
5. 调用onComplete回调返回三个路径
```

### 状态管理

```dart
bool _isTestMode           // 当前是否在测试模式
bool _isRecording          // 是否正在录制（仅在即时模式下使用）
RecordingHistoryEntry? _selectedTestVideo  // 选中的测试视频
```

  - 视频名称
  - 时长和Round信息
  - "选择"按钮

## ✅ 使用提示

### ✓ 最佳实践
- **提前导入**：在使用测试模式前，确保已有足够的测试视频
- **分类命名**：为测试视频使用清晰的名称，便于快速查找
- **保留原始**：建议保留原始视频，用于多次测试

### ⚠️ 注意事项
- 测试模式生成的CSV仅包含头部（无实际的姿态数据）
- 如需完整的姿态分析，仍需使用即时录制模式或上传到后端处理

## 🐛 故障排查

### 问题：找不到已导入的视频
**原因**：视频文件可能已被删除或移动
**解决**：检查应用数据目录中的视频文件是否存在

### 问题：测试模式下完成后没有反应
**原因**：可能是文件复制期间出现错误
**解决**：检查应用是否有足够的磁盘空间，查看调试日志
