**流程：** 完全正確且高效

```

    ↓

    ↓

    ↓

    └─ cv2.VideoWriter 單次編碼 (mp4v codec)
    ↓

  ✅ pose_landmarks.csv (正確的座標和可見度)
  ✅ skeleton_overlay.mp4 (骨架視頻，單次編碼)
```

**優點：**
- ✅ 單次編碼（質量損失最小）

## 🔍 座標系統驗證

### 座標流向（正確）

```
Python 端：
  MediaPipe 標準化座標 [0,1]
    × 旋轉後視頻尺寸 (w, h)
    = 像素座標 (xPx, yPx) in rotated space
    ↓ 寫入 CSV

Kotlin 端：
  CSV 座標 (xPx=960, xNorm=0.5)
    ÷ 標準化座標
    = 原始檢測空間尺寸 (1920)
    ÷ 原始檢測空間尺寸
    = 縮放因子 (1920/1920 = 1.0)
    ↓ 應用到繪製
    = 正確的像素位置 ✅
```

**結論：** 坐標系統邏輯上完全正確 ✅
