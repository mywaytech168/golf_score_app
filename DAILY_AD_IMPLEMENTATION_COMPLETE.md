## 💾 LocalStorage 設計

使用 SharedPreferences 存儲廣告狀態：

```dart
// 最後一次看廣告的日期 (YYYY-MM-DD 格式)
daily_ad_watched_date: "2026-03-24"

// 是否在今天已使用過廣告機會
daily_ad_used_today: true/false
```

**優勢**:
- ✅ 輕量級，無需數據庫
- ✅ 自動持久化
- ✅ 應用卸載時清空
- ✅ 支持日期判斷自動重置

## 🧪 使用場景演示

### 場景 1：完整流程（第一次看廣告）
```
時間: 2026-03-24 上午
1. 點擊「錄影」按鈕
2. ↓ showAdCheckDialog() 被觸發
3. ↓ 檢查：未看過廣告
4. ↓ 彈出「看廣告玩」和「購買」選項
5. 用戶選擇「看廣告玩」
6. ↓ 播放獎勵廣告
7. 用戶看完廣告
8. ↓ DailyAdManager.markAdAsUsed()
9. ↓ 對話框關閉，調用 onContinue()
10. ✅ 進入錄影畫面
11. ↓ 保存狀態到 SharedPreferences
```

### 場景 2：同一天再次點錄影
```
時間: 2026-03-24 下午
1. 點擊「錄影」按鈕
2. ↓ showAdCheckDialog() 被觸發
3. ↓ 檢查：已看過廣告
4. ✅ 直接進入錄影畫面（無彈窗！）
```

### 場景 3：中途回到首頁，再次點錄影
```
時間: 2026-03-24 中途
1. 用戶在錄影中按返回鍵回到首頁
2. 狀態保持: daily_ad_used_today = true
3. 再次點「錄影」
4. ↓ showAdCheckDialog() 檢查
5. ↓ 日期相同，狀態保留
6. ✅ 直接進入錄影（無彈窗！）
```

### 場景 4：隔天自動重置
```
時間: 2026-03-25 早上
1. 點擊「錄影」按鈕
2. ↓ showAdCheckDialog() 被觸發
3. ↓ DailyAdManager.hasUsedAdToday()
4. ↓ _checkAndResetDaily() 檢查日期
5. ↓ 日期改變："2026-03-24" → "2026-03-25"
6. ↓ 🔄 自動重置 daily_ad_used_today = false
7. ↓ 彈出廣告對話框
8. 用戶又可以看一次廣告
```

## 🐛 調試功能

### 測試按鈕位置
首頁 AppBar 右上方（需要在調試模式運行）：
- **🧪 購買測試**: 測試應用內購買
- **🔄 重置廣告**: 快速重置廣告使用狀態

### 使用測試按鈕
```
1. 在調試模式下運行應用
2. 看完廣告進入錄影
3. 回到首頁，點 🔄 重置廣告
4. SnackBar 顯示: "✅ 廣告使用狀態已重置"
5. 再次按錄影 → 廣告對話框重新出現
```

```

✅ [廣告] 已標記用戶使用了今天的廣告機會
📺 [廣告] 今天是否已使用過廣告機會: true
✅ [廣告] 用戶今天已使用過一次廣告，直接進入錄影（無彈窗）
🔄 [廣告] 已重置廣告使用狀態（測試用）
```
