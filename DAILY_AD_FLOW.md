## 一次性廣告機制流程圖

### 用戶按下「錄影」或「IMU 按鈕」

```
用戶操作 → showAdCheckDialog() 被觸發
    ↓
[檢查 1] 是否是高級用戶？
    ├─ YES → ✅ 直接進入錄影（無彈窗）
    └─ NO ↓
       [檢查 2] 今天是否已使用過廣告機會？
           ├─ YES → ✅ 直接進入錄影（無彈窗）
           └─ NO ↓
              [彈出廣告選擇對話框]
              用戶選擇：
              ├─ 「看廣告玩」
              │   ↓
              │   📺 播放獎勵廣告
              │   ↓
              │   [用戶看完廣告]
              │   ↓
              │   DailyAdManager.markAdAsUsed()
              │   ↓
              │   ✅ 進入錄影（關閉對話框，調用 onContinue()）
              │
              ├─ 「購買無廣告版本」
              │   ↓
              │   💳 開啟應用商店購買流程
              │   ↓
              │   [購買成功]
              │   ↓
              │   ✅ 進入錄影
              │
              └─ 「取消」
                  ↓
                  ❌ 留在當前頁面（不進入錄影）
```

### 隔天自動重置

```
隔天用戶進入 App
    ↓
任何調用 DailyAdManager 的方法
    ↓
_checkAndResetDaily() 檢查日期
    ↓
[日期是新的一天？]
    ├─ YES → 🔄 重置廣告使用狀態
    │        日期已更新
    └─ NO → 保持當前狀態
```

### 狀態機圖

```
┌─────────────────────────────────────────────┐
│          DailyAdManager 狀態機              │
└─────────────────────────────────────────────┘

  [新建/重置]
      ↓
      ▼
┌──────────────────────────┐
│ 今天未使用廣告機會       │  ← hasUsedAdToday() = false
│                          │
│ 第一次按錄影 → 彈出廣告  │
│ 用戶可選擇看廣告或購買   │
└──────────────────────────┘
      ↓ [看完廣告後]
      │ markAdAsUsed()
      ↓
┌──────────────────────────┐
│ 今天已使用廣告機會       │  ← hasUsedAdToday() = true
│                          │
│ 再按錄影 → 直接進入      │
│          （無彈窗）       │
│ 回到首頁 → 狀態保留      │
└──────────────────────────┘
      ↓ [隔天]
      │ _checkAndResetDaily()
      │ 日期改變 → 重置狀態
      ↓
┌──────────────────────────┐
│ 新的一天 - 恢復初始狀態   │
│                          │
│ 第一次按錄影 → 彈出廣告  │
└──────────────────────────┘
```

### SharedPreferences 字段映射

| 字段名稱 | 類型 | 說明 | 示例 |
|---------|------|------|------|
| `daily_ad_watched_date` | String | 最後一次記錄廣告日期 | `"2026-03-24"` |
| `daily_ad_used_today` | bool | 是否在今天已使用過廣告機會 | `true` / `false` |

### 調試按鈕操作

```
首頁 AppBar 右上方調試按鈕（僅在 kDebugMode 下顯示）
    ↓
[🧪 購買測試] → 打開購買測試面板
[🔄 重置廣告] → 重置廣告使用狀態
               ↓
               DailyAdManager.resetAdUsage()
               ↓
               daily_ad_used_today = false
               ↓
               SnackBar: "✅ 廣告使用狀態已重置"
```

### 用戶場景演示

#### 場景 1：完整流程（第一次看廣告）

```
時間: 2026-03-24 10:00
用戶進入首頁 → 點擊「錄影」
    ↓
showAdCheckDialog() 檢查
    - 高級用戶? NO
    - 今天用過? NO → 彈出對話框
    ↓
用戶點「看廣告玩」
    ↓
播放獎勵廣告
    ↓
用戶看完廣告
    ↓
DailyAdManager.markAdAsUsed()
    - 設置 daily_ad_used_today = true
    - 設置 daily_ad_watched_date = "2026-03-24"
    ↓
✅ 進入錄影畫面
```

#### 場景 2：同一天再次點錄影

```
時間: 2026-03-24 14:30
用戶回到首頁 → 再次點擊「錄影」
    ↓
showAdCheckDialog() 檢查
    - 高級用戶? NO
    - 今天用過? YES → 直接進入
    ↓
✅ 進入錄影畫面（沒有彈窗）
```

#### 場景 3：隔天自動重置

```
時間: 2026-03-25 10:00
用戶進入 App → 點擊「錄影」
    ↓
showAdCheckDialog() 檢查
    - _checkAndResetDaily() 檢查日期
    - 上次日期: "2026-03-24"
    - 今日日期: "2026-03-25"
    - 日期不同 → 🔄 重置狀態
    ↓
hasUsedAdToday() 檢查
    - daily_ad_used_today = false（已重置）
    ↓
彈出廣告對話框（新的一天，又可看一次）
    ↓
用戶可以再看一次廣告
```

#### 場景 4：中途離開並返回（狀態保留）

```
時間: 2026-03-24 10:00
用戶看完廣告進入錄影
    ↓
保存狀態:
    - daily_ad_used_today = true
    - daily_ad_watched_date = "2026-03-24"
    ↓
用戶中途按返回鍵回到首頁
    ↓
時間: 2026-03-24 10:30
用戶再次進入首頁 → 點擊「錄影」
    ↓
showAdCheckDialog() 檢查
    - daily_ad_used_today = true（狀態保留）
    ↓
✅ 直接進入錄影（無彈窗）
    ↓
※ 使用狀態沒有被重置，因為仍是同一天
```
