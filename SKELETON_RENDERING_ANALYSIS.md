## 📐 尺寸計算

### 線條寬度 (strokeW)

```kotlin
val strokeW = (w / 120f).coerceIn(3f, 9f)
```

**含義**：根據視頻寬度自適應

| 視頻寬度 | 線條寬度 |
|----------|---------|
| 360px | 3.0px (最小) |
| 1080px | 9.0px (最大) |
| 1920px | 9.0px (鉗制上限) |

**特點**：
- ✅ 自動適應不同解析度
- ✅ 最小 3px（不會太細），最大 9px（不會太粗）

### 圓點半徑 (radius)

```kotlin
val radius = (w / 90f).coerceIn(4f, 12f)
```

**含義**：根據視頻寬度自適應

| 視頻寬度 | 圓點半徑 |
|----------|---------|
| 360px | 4.0px (最小) |
| 1080px | 12.0px (最大) |
| 1920px | 12.0px (鉗制上限) |

**特點**：
- ✅ 圓點相對線條更大（視覺凸顯）
- ✅ 自動適應不同解析度

## 👁️ 可見性過濾

### 置信度閾值

```kotlin
if (la.visibility < 0.3f || lb.visibility < 0.3f) continue  // 線條
if (lm == null || lm.visibility < 0.3f) continue            // 圓點
```

**閾值 = 0.3f (30%)**

- ✅ **> 0.3f**：繪製（可信度高）
- ❌ **≤ 0.3f**：略過（可信度低或遮擋）

**效果**：
- 只顯示模型有信心的關鍵點
- 避免繪製遮擋或不可見的部位
- 減少視覺雜亂

## 📍 座標轉換

### 從 CSV 空間到視頻空間

```kotlin
val scaleX = videoW.toFloat() / poseW
val scaleY = videoH.toFloat() / poseH

// 繪製時應用縮放
canvas.drawLine(
    la.xPx * scaleX, la.yPx * scaleY,
    lb.xPx * scaleX, lb.yPx * scaleY,
    linePaint
)
```

**過程**：

1. **CSV 中的座標** (`xPx`, `yPx`)：基於縮圖空間（通常 720px 寬）
2. **推算 poseW, poseH**：從 CSV 數據的標準化座標反推
3. **計算縮放比** (`scaleX`, `scaleY`)：`實際視頻寬高 / 縮圖寬高`
4. **應用縮放**：`位置 × 縮放比 = 實際視頻中的位置`

**例**：
- 視頻 1920×1080，縮圖 720×720
- scaleX = 1920 / 720 ≈ 2.67
- scaleY = 1080 / 720 ≈ 1.5
- CSV 中 (100, 100) → 視頻中 (267, 150)

## 💡 視覺設計特點

### 優點 ✅

1. **顏色編碼清晰**：
   - 左側綠色，右側紅色
   - 中央部位黃色
   - 符合人體解剖學標準

2. **自適應尺寸**：
   - 根據解析度自動調整線寬和點大小
   - 360p 到 4K 都能看清

3. **可見性過濾**：
   - 只顯示可信度 > 30% 的點
   - 避免虛假或遮擋的部位

4. **視覺層次**：
   - 圓點大於線條寬度
   - 點顏色比線條深
   - 容易定位關鍵點

5. **完整的人體覆蓋**：
   - 33 個點涵蓋全身
   - 臉、四肢、躯幹、腿都有

### 改進空間 ⚠️

1. **沒有標籤**：
   - 看不出哪個點是什麼部位
   - 需要畫面外文字標註

2. **沒有骨骼粗細變化**：
   - 所有線條寬度相同
   - 不能強調重要關節（如髖、肩）

3. **沒有深度提示**：
   - 前後遮擋關係不明顯
   - 適合正面視圖

4. **沒有動作信息**：
   - 單幀無法看出運動方向
   - 需要視頻連續播放才能感受

## 🚀 建議的增強方向

如果需要改進骨架可視化：

1. **添加標籤**：在關鍵點旁顯示部位名稱
2. **動態線寬**：主要關節線條更粗
3. **置信度視覺化**：低置信度用淡色或虛線
4. **骨骼厚度**：根據身體部位（腿比臂粗）
5. **動作箭頭**：显示關鍵點的運動方向
6. **多幀軌跡**：尾跡效果顯示運動路徑
