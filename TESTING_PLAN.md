### 測試 3: 解析度驗證

```
目標：確認 VideoTrimmer 是否保留解析度
方法：使用 ffprobe 檢查
步驟：
  1. 獲取原始視頻：
     ffprobe -v error -select_streams v:0 \
       -show_entries stream=width,height,r_frame_rate \
       -of csv=s=x:p=0 original_video.mp4
     
     預期輸出：1920x1080,30/1
  
  2. 應用 VideoTrimmer 裁切後：
     ffprobe -v error -select_streams v:0 \
       -show_entries stream=width,height \
       -of csv=s=x:p=0 trimmed_video.mp4
     
     預期輸出：1920x1080  (相同！)
```

## 🔍 潛在問題診斷

### 問題 1: 骨架位置不對齐

**症狀：** 骨架出現在視頻的錯誤位置

**診斷方法：**
1. 檢查日誌中 "骨架影像尺寸推算" 的值（應為原始解析度）
2. 檢查 rotation 值是否與視頻元數據一致
3. 使用 ffprobe 驗證各層解析度

### 問題 2: 座標顛倒（旋轉視頻特定）

**症狀：** 旋轉視頻中骨架方向錯誤

**診斷方法：**
1. 確認 codedToDisplay 只用於球軌跡檢測
2. 測試無旋轉視頻是否正常

### 問題 3: 質量沒有改善

**症狀：** 最終視頻仍然模糊

**診斷方法：**
1. 檢查日誌中 bitRate 值（預期：25 Mbps）
2. 驗證檔案大小（新方案應更大）

### 問題 4: 最後幾幀丟失

**症狀：** 視頻在結尾停止

**診斷方法：**
1. 檢查 EOS 信號日誌
2. 驗證 encodedFrames == samplesWritten
